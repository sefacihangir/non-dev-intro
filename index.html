Development Introduction

For Non-Developers

by Serban Petrescu

Our Goals

  • To see what development is all about.
  • To clear up some misconceptions.
  • To understand the specific reasoning and decision-making.

Not Our Goals

  • To learn how to code.
  • To go (too) deep into technical topics.

Content

We will build a very simple web application and pass through all the typical stages:

  • Business requirement
  • Analysis and design
  • Development
  • Documentation

"Rules"

At each stage, we will discuss together about what has to be done.

  • Ask any question related to the current topic as soon as you have the chance
  • Any other general questions should be asked at the end

Overview

The app will manage a list of employees. It will be deployed to the SAP Cloud Platform and will use the following technologies:

  • Java 8
  • JPA
  • JAX-RS
  • SAP UI5

Business Requirements

Each project should have a business specification (the goals and reasoning).

Then each concrete requirement should be described.

The requirements can be either functional (what) or non-functional (how).

Purpose

We aim to digitalize the management of our employees by creating an online-tool for storing, retrieving and processing employee data.

Scope

The new solution will work solely with the data related to current and past employees.

The data itself consists of:

  • Full name.
  • Current role.
  • Hire date.

Functional

When opening the application, a table of employees should be displayed. The table should have two buttons in its header:

  • A refresh button. When pressed, the table contents should be reloaded.
  • An add button. When pressed, the “Create Employee” dialog should be opened.

Functional Cont'd

After pressing the Confirm button of the “Create Employee”, a new employee with the filled in properties should be created.

The table should be automatically refreshed to show the new employee.

Mockup

Non-Functional

The application should be accessible through a browser, with no additional software installation needed.

The expected number of concurrent users is in the range 1 – 20.

Analysis and design

A technology stack must be chosen such that we can fulfill the requirements.

Afterwards, the system architecture and high level design must be engineered.

These will act as blueprints for the development team.

Analysis

  • Data has to be persisted centrally.
       → Database
  • Application needs to be accessible without installation.
       → HTTP Server
  • UI has to work inside a browser.
       → HTML + JavaScript
  • We don’t have any dedicated infrastructure.
       → Cloud

Technologies

Cloud: SCP. It takes care of the database and provides a "HTTP Server" (Tomcat).

Because of Tomcat, we will use server-side Java.

UI: SAP UI5 (can also use e.g. Angular).

Architecture

Design

Design

  • The UI will also be shipped within the server application.
  • When the user opens up the app:
    • The server will redirect him to the index.html.
    • The index.html will load the UI5 libraries.
    • The UI5 libraries will load the view + controller.
  • The controller will then communicate with the server.

Implementation

Each developer works on his local environment. Specialized software (git, scm) is used for collaboration and versioning.

Periodically the shared environment is updated with the local changes of each developer.

Manual / automatic tests should also be written.

Local

First we will implement the application in our local environment. We will build it using Eclipse and test it on a local Tomcat installation.

Cloud

Then we will deploy and test it on the SCP.

The local development is setup to mimic the cloud environment.
